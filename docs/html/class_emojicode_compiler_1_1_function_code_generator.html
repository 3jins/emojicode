Detailed Description

A FunctionCodeGenerator instance is responsible for generating the LLVM IR for a single function.

It manages the declaration of arguments, the context of the function, the scoper and is responsible for keeping track of temporary objects.

This class provides helper methods to simplify use of the LLVM API and for common actions inside the compiler’s code base, as for example retrieving the class info from an object.

◆ stackAlloc()

llvm::Value * EmojicodeCompiler::FunctionCodeGenerator::stackAlloc ( llvm::PointerType *  type)

Allocates stack memory as replacement for a heap memory allocation as performed by alloc().

In order to ensure compatibility with the runtime library’s retain and release functions, additional bytes are allocated in front of the object.

Note
Like ejcAlloc, this function expects the first element of the allocated type to be a pointer to the control block.
